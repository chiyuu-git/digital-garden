## 优化总结

常用函数会被编译成机器码，执行效率远高于字节码

那 util 的存在就很重要了，只要类型完全确定就可以去做这个优化，不必考虑类型的改变导致需要重新编译

如果类型发生了改变，只能重新编译一次字节码生成新的机器码

字节码抽象的汇编语言，比较类似

v8 定义了一大堆字节码，v8 都有说明其作用

解释器：代码 -> ast -> 字节码

编译器：字节码 -> 机器码

JIT，真正用到函数的时候才决定是否要编译这个函数
